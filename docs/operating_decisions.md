# Operating Decisions

長期運用で迷いやすい点を先に固定する。世界観そのものではなく、制作管理の決めごと。

## 採用ステータス

`data/uragawa_ledger.tsv` の `status` は次のいずれかにする。

- `idea`: 種だけある。通常は `data/idea_bank.tsv` で管理し、台帳には入れない
- `draft`: 観測文とビジュアル設計まで作った
- `image_prompted`: 画像生成プロンプトまで作った
- `image_reviewed`: 生成画像をレビューした
- `sns_test`: SNS投稿候補
- `adopted`: 正式採用
- `merch_candidate`: 商品化候補
- `merch_ready`: 商品化用に再編集済み
- `retired`: 反応や品質上の理由で休止

正式な `No.001` 形式は `adopted` 以降にだけ使う。試作段階は `No.XXX` とする。

## 番号ルール

- 台帳の `id` は `No.001` の3桁形式にする。
- 番号は採用順で、世界観上の強さや重要度を表さない。
- 欠番は許容する。削除や差し戻しで番号を詰め直さない。
- 次の番号は `python scripts/next_uragawa_number.py` で確認する。

## 反応方向の偏り管理

直近10件で同じ反応方向が5件以上続く場合は、次回の制作で意識的に別方向を試す。

例:

- かわいいが多いなら、少し怖い、変で忘れられない、妙に納得へ振る。
- エモいが多いなら、取り扱いメモ、注意書き、クスッと笑える方向へ振る。
- 怖いが多いなら、おしゃれ、詩的、日用品として持ちたい方向へ振る。

## 視覚トリックの偏り管理

直近で同じ演出が続いたら避ける。

- 中に何かがいる
- 影だけ違う
- 反射だけ違う
- 断面に秘密がある
- めくれ
- 収納
- 糸、根、光、湯気、泡でつながる
- 置かれ方で感情が出る

繰り返す場合は、関係性やシリーズ性として意図を明記する。

## 採用の最低条件

- 現実との接続点が1つ以上、画面内で見える。
- 裏事情が文章だけでなく、画像プロンプトにも入っている。
- 文字なしの商品化可否が判断されている。
- `docs/motif_suitability_gate.md` と `data/motif_taxonomy.tsv` に照らして、主役題材としての適性がある。
- `docs/product_format_rules.md` と `docs/product_qa.md` に照らして、横長専用ではない。
- 台帳登録用の1行がある。
- `scripts/validate_uragawa_output.py --strict` が通る。

## 商品化候補の最低条件

- 小さく印刷しても外形が読める。
- 文字なしでも成立するか、文字を入れる理由がある。
- 白背景または透明背景に置ける。
- Tシャツ、ステッカー、スマホケース、トートの少なくとも1つで具体的な置き方がある。
- SNS投稿時の説明を削っても、所有したくなる視覚的な魅力が残る。
