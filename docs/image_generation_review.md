# Image Generation Review

gpt-image2.0で画像を生成したあとに見る観点。画像そのものを保存する場合は `assets/generated/` か、制作単位のフォルダを作って管理する。

## 1. モチーフ確認

- 元のモチーフが一目でわかるか。
- 単純化しすぎて別物になっていないか。
- 重要な現実との接続点が残っているか。

## 2. 裏事情確認

- 「普通と違う」要素が画面内に見えるか。
- 裏事情がキャプションだけに依存していないか。
- 視線が裏事情の核へ自然に向くか。

## 3. 商品化確認

- 外周シルエットがステッカー向きか。
- Tシャツに置いたときに主張が強すぎないか。
- スマホケースで縦長配置にできるか。
- トートバッグで中央または下部ワンポイントとして成立するか。
- 小サイズでも潰れないか。

## 4. 修正判断

### プロンプト再生成で直す

- 裏事情が弱い
- 余白が少ない
- 主線や色が重い
- モチーフがわかりにくい
- 商品化に向かない構図

### アイデアから作り直す

- 画像化すると普通のキャラクターにしかならない
- 現実との接続が絵に残らない
- 過去作と視覚トリックがほぼ同じ
- 説明がないと意味が成立しない

## 5. 記録

画像レビューは `templates/image_review.md` を使う。

良い失敗も記録する。特に、画像生成では崩れやすい表現、効いた表現、商品化で強く見えた構図は `data/continuity_notes.md` に追記する。
