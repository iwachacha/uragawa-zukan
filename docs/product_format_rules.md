# Product Format Rules

SUZURIの商品化を前提にする場合、最初から正方形または縦長でも成立する設計にする。
詳細な公式仕様は `docs/suzuri_product_references.md` を参照する。正式入稿前は必ずSUZURI公式テンプレートを再確認する。

## 優先フォーマット

1. **Square primary**: 正方形。ステッカー、Tシャツ中央、トート、スマホケースに展開しやすい。
2. **Vertical secondary**: 縦長。スマホケース、Tシャツ中央、トートに向く。
3. **One-point**: 小さな単体。左胸、袖、端配置に向く。
4. **Pattern**: 総柄。小さな要素が複数あっても魅力が落ちない場合。
5. **Horizontal sticker only**: 横長ステッカー専用。原則として商品化優先度は下げる。

## 避ける構図

- 横長にしないと成立しない行列。
- 背景込みでないと意味が伝わらない構図。
- 主役が小さく、余白に負ける構図。
- 商品に置いたとき端が切れる長い線や紐。
- 現実スケールを大きく崩して、元モチーフがわからなくなる構図。

## プロンプトに入れる指定

画像生成プロンプトには、以下を必ず書く。

- `primary format: square`
- 必要なら `secondary use: vertical phone case / one-point / sticker`
- 主役と周辺物の相対スケール
- 背景なしでも成立すること
- 横長専用にしないこと

## 商品化判断

商品化候補は、最低でも次の2つに展開できること。

- ステッカー
- Tシャツまたはトート
- スマホケース
- ワンポイント
- 総柄

## 推奨マスター設計

- 通常候補: 3000 x 3000 px 以上の正方形、透過PNG想定。
- 細密な世界観候補: 4000 x 4000 px 以上で作り、商品別に縮小・再配置する。
- 縦長展開候補: 3000 x 4500 px 程度の縦長再構成案も作る。
- ステッカー候補: 630 x 630 pxに縮小しても読めるか確認する。
- アクリル候補: 細い突起、離れた小パーツ、3px以下の線や文字を避ける。
